# ADR 0002 — Шарова архітектура

## Структура
Проєкт використовує 3-шарову архітектуру:

- api/ — HTTP-рівень, контролери та роутинги
- service/ — бізнес-логіка, use-cases
- domain/ — доменні сутності, інваріанти

## Правила залежностей
api -> service -> domain

## Заборонено
❌ api напряму працює з БД  
❌ domain не знає про http  
❌ перехресні залежності між шарами

## Статус
Прийнято
